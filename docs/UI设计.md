# UI 设计规范

## 整体布局

- **全屏 Canvas 渲染**：所有游戏画面通过 Canvas 2D API 绘制，无 DOM 元素叠加（除开始/结束界面）
- **画布尺寸**：800 × 600 px（`CONFIG.CANVAS_WIDTH × CONFIG.CANVAS_HEIGHT`）
- **背景**：深空黑色 `#0a0a0a`，带动态星空粒子背景

## HUD（抬头显示）

始终在 Canvas 顶部和底部绘制，不遮挡游戏主区域：

| 元素 | 位置 | 内容 | 说明 |
|------|------|------|------|
| 分数 | 左上角 | `分数: N` | 白色字体，Arial 16px |
| 生命值 | 左上角（分数下方） | `❤️ × N` 或 `HP: N` | 用红心符号或文字显示 |
| 关卡 | 右上角 | `第 N 关` | 白色字体 |
| 道具指示器 | 右下角 | 散弹图标「S」 | 有道具时高亮，可点击切换 |
| Boss HP 条 | 顶部居中 | 血量条 + Boss名字 | 仅Boss战时显示 |
| 敌人数 | 右上角（关卡下方） | `剩余: N/M` | 显示本关剩余敌人数 |

## 游戏状态界面

### 开始界面
- **标题**：`太空战机` 居中大号白色字体
- **按钮**：「开始游戏」—— 居中绿色按钮
- **背景**：动态星空 + 粒子效果，预览游戏画面

### 过渡动画（关卡切换）
1. **阶段1（淡入）**：玩家战机飞向屏幕中心（800ms）
2. **阶段2（停留）**：玩家向上飞出屏幕（600ms）
3. **阶段3（黑屏）**：黑色半透明遮罩，居中显示「下一关：xxx」（1500ms）

### Game Over 界面
- **标题**：`游戏结束` 红色居中
- **信息**：显示最终得分
- **按钮**：「重新开始」

### 胜利界面
- **标题**：`恭喜通关！` 金色居中
- **信息**：最终得分 + 击杀数
- **按钮**：「再来一局」

## 视觉风格

### 配色

| 角色 | 颜色 | 色值 |
|------|------|------|
| 玩家战机 | 蓝色 | `#4488ff`（主体）、`#66aaff`（高光） |
| 引擎火焰 | 橙黄渐变 | `#ff6600` → `#ffaa00` |
| 玩家子弹 | 白色/淡蓝 | `#ffffff` / `#88ccff` |
| 黄色敌人 | 金黄 | `#ffcc00` 背景 + 黑色瞄准 |
| 绿色追踪敌 | 绿色 | `#44ff44` 箭头 |
| Boss | 红色 | 红色主体 + 渐变 |
| Boss 子弹 | 红色 | `#ff4444` |
| 道具 (散弹) | 蓝色 | `#4488ff` |
| 爆炸粒子 | 黄色 | `#ffff00` |
| 追踪弹 | 白色 | `#ffffff` |
| 粒子（通用） | 随机 | 从 `randomColor` 生成 |

### 字体
- 全部使用 Canvas 原生文字渲染
- 默认字体：`Arial`（等宽/非等宽混合）
- 标题字号：32px bold
- HUD 字号：16px
- 过渡动画标题：32px bold

### 粒子效果

| 类型 | 说明 | 粒子数 | 颜色 |
|------|------|--------|------|
| 爆炸（黄色敌死亡） | 放射状扩散 | 20 | `#ffff00` |
| 闪光（绿色敌死亡） | 十字星抖动 | 8 | `#44ff44` |
| Boss 爆炸 | 3层同心圆 + 白色高亮 | 90+15 | 随机彩色 |
| 引擎尾焰 | 持续尾迹（代码drawPlayer中绘制） | — | 橙黄渐变 |

## DOM 元素

仅以下元素由 HTML 提供（覆盖在 Canvas 之上）：

- 开始按钮（`#start-btn`）
- 重新开始按钮（`#restart-btn`、`#win-restart-btn`）
- 道具指示器（`#spread-indicator`）
- 各种状态面板（`.screen` 系列）

所有 DOM 元素通过 CSS `position: absolute` 定位到 Canvas 容器上。
